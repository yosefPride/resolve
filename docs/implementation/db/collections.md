# Collections — Field Reference

Every field of every collection that exists in code, with its BSON type, the Rust struct it
maps to, and who writes it.

---

## `users`

**Rust:** `user::models::User` · **Repository:** `UserRepository` (`src/user/repository.rs`)

| Field | BSON type | Notes |
|---|---|---|
| `_id` | ObjectId | Generated by Mongo on insert. |
| `email` | String | **Unique index.** Matched case-sensitively (`find_by_email` is an exact match), so `Ada@x.com` and `ada@x.com` are different accounts. |
| `password_hash` | String | bcrypt, work factor 12. Never leaves the backend — `UserResponse` omits it. |
| `name` | String | Display name. |
| `global_role` | String \| null | `"SystemAdmin"` or absent/null. **No code path sets this** — `UserRepository::create` always writes `None`. Must be set directly in the database. Note this enum has *no* serde rename, so it's PascalCase, unlike every other enum in the schema. |
| `created_at` | Date | Set to `now` on insert. |

**Written by:** `create` (register), `update_profile` (name/email), `update_password_hash`,
`delete` (admin user deletion).

**Read by:** login, `GET /auth/me`, `RbacService::require_system_admin`, admin user list,
and the name-join in `enrich_member` / `enrich_ticket`.

## `refresh_tokens`

**Rust:** `auth::models::RefreshTokenDoc` · **Repository:** `AuthRepository`

| Field | BSON type | Notes |
|---|---|---|
| `_id` | ObjectId | |
| `user_id` | ObjectId | → `users._id`. |
| `token_hash` | String | **Unique index.** SHA-256 hex of the raw token. The raw token exists only in the client's cookie and is never stored — a leaked database cannot mint sessions. |
| `created_at` | Date | |
| `expires_at` | Date | `now + 30 days`. **TTL index** (`expireAfterSeconds: 0`), so Mongo deletes the row automatically once this passes. |
| `revoked_at` | Date \| null | `null` while active. Set on rotation (refresh), logout, or password change. |

Session data, **not tenant data** — no `group_id`, and never touched by group-scoped logic.

A row is effectively single-use: `find_active_by_hash` requires `revoked_at: null` **and**
`expires_at > now`, so a replayed or stale token isn't found. Rows are also physically reaped
by the TTL index, meaning the collection self-limits to roughly the number of live sessions.

## `groups`

**Rust:** `group::models::Group` · **Repository:** `GroupRepository` (`groups` field)

| Field | BSON type | Notes |
|---|---|---|
| `_id` | ObjectId | The tenant identifier. Appears in every group-scoped URL. |
| `name` | String | Validated non-blank; **no maximum length**. Not unique — two groups may share a name. |
| `owner_id` | ObjectId | → `users._id`, the creator. **Informational only.** No query filters on it and no authorization reads it; permissions come from `group_members.role`. |
| `created_at` | Date | |

**No secondary indexes** — nothing queries groups by anything but `_id`, except the admin
list which is an unindexed scan with an optional name regex.

## `group_members`

**Rust:** `group::models::GroupMember` · **Repository:** `GroupRepository` (`members` field)

The RBAC table and the many-to-many join between `users` and `groups`.

| Field | BSON type | Notes |
|---|---|---|
| `_id` | ObjectId | |
| `group_id` | ObjectId | → `groups._id`. |
| `user_id` | ObjectId | → `users._id`. |
| `role` | String | `"contributor"` \| `"group_admin"` (snake_case serde rename). **This is the only place a group role is stored** — never in the JWT, never on the user. |
| `joined_at` | Date | |

**Unique compound index on `(group_id, user_id)`** — one row per pair. This is what makes
duplicate-membership rejection atomic: `insert_member` inserts optimistically and maps error
11000 → `GroupRepoError::DuplicateMember` → `409`, instead of a racy check-then-insert.

**Invariant not enforced by the database:** every group must have at least one row with
`role: "group_admin"`. Upheld only by `GroupService::guard_sole_admin_removal`, which runs on
member removal, self-leave, and demotion. Group deletion deliberately bypasses it.

## `tickets`

**Rust:** `ticket::models::Ticket` · **Repository:** `TicketRepository` (`tickets` field)

| Field | BSON type | Notes |
|---|---|---|
| `_id` | ObjectId | Internal id, used in URLs. |
| `group_id` | ObjectId | → `groups._id`. **Present in every single query's filter**, including single-document reads. |
| `ticket_number` | Int64 | The human-facing number, 1, 2, 3… **per group**. Allocated atomically from `counters`. **Unique compound index on `(group_id, ticket_number)`.** |
| `title` | String | Non-blank, ≤ 200 **bytes** (`title.len()`, not chars). |
| `description` | String | Non-blank, no length cap. |
| `status` | String | `"open"` \| `"closed"`. Always `"open"` on insert — never accepted from the client. |
| `priority` | String | `"low"` \| `"high"` \| `"critical"`. **No `"medium"`.** Supplied at creation. |
| `created_by` | ObjectId | → `users._id`, taken from the token, never from the request body. |
| `created_at` | Date | |
| `updated_at` | Date | Set to `created_at` on insert; refreshed on every `PATCH` (always, regardless of which fields changed). |

**No `assignee` field** — tickets are never assigned to anyone. This is a product decision,
not an omission.

`created_by` is a dangling reference after that user is deleted; `enrich_ticket` renders
`created_by_name` as `""` rather than failing.

## `counters`

**Rust:** `ticket::models::TicketCounter` · **Repository:** `TicketRepository` (`counters` field)

| Field | BSON type | Notes |
|---|---|---|
| `_id` | ObjectId | **Is the `group_id`.** Not a separate generated id — this gives a natural 1-to-1 with `groups` plus free uniqueness. |
| `ticket_seq` | Int64 | Last-allocated `ticket_number` for this group. |

The only collection with a non-generated `_id`, and the only one whose Rust struct declares
`#[serde(rename = "_id")]` on a **non-optional** field.

Written exclusively by `next_ticket_number`:
```rust
find_one_and_update({_id: group_id}, {$inc: {ticket_seq: 1}})
    .upsert(true).return_document(After)
```
`upsert` creates the document on a group's first ticket (yielding 1). A single atomic
server-side operation, so concurrent creations in one group cannot collide.

Never deleted — a group's counter survives the group. See [`../deviations.md`](../deviations.md).

## `admin_audit_log`

**Rust:** `admin::models::AuditLogEntry` · **Repository:** `AdminRepository`

| Field | BSON type | Notes |
|---|---|---|
| `_id` | ObjectId | |
| `action` | String | `"succession"` \| `"group_auto_deleted"`. |
| `group_id` | ObjectId | **Indexed.** The group whose admin structure changed. |
| `group_name` | String | **Snapshot.** `#[serde(default)]`. |
| `deleted_user_id` | ObjectId | **Indexed.** The user being deleted. |
| `deleted_user_name` | String | **Snapshot.** `#[serde(default)]`. |
| `successor_user_id` | ObjectId \| null | Set only when `action = "succession"`. |
| `successor_user_name` | String \| null | **Snapshot.** `#[serde(default)]`. |
| `performed_by` | ObjectId | The System Admin who ran the deletion. |
| `performed_by_name` | String | **Snapshot.** `#[serde(default)]`. |
| `created_at` | Date | Sorted descending on read. |

### Why the name snapshots

This is the one place the schema deliberately denormalizes. By read time, the deleted user
**always** no longer exists, and for `group_auto_deleted` the group doesn't either — so
their ids cannot be resolved to names after the fact. Storing the name alongside the id at
write time is the only way the log stays readable.

`#[serde(default)]` on each `*_name` means entries written before those fields existed still
deserialize (reading back with an empty name) rather than failing the entire query. That's a
schema-migration accommodation encoded in the type.

The frontend leans on this: `AuditLogPanel` builds its filter dropdowns from the log's own
snapshotted names, precisely because a lookup against `/admin/users` or `/admin/groups`
couldn't resolve deleted entities.

**Append-only** — nothing updates or deletes entries.

**Not written for `DELETE /admin/groups/{id}`.** Only succession and auto-deletion are
logged; an admin deleting a group outright leaves no audit trail.

## Specified but not implemented

These appear in `docs/specification/database.md` and have **no Rust type, no repository, no
index, and no code that touches them**. The `src/comment/` and `src/ai/` modules are empty files.

- `comments` — `_id, group_id, ticket_id, user_id, content, created_at`
- `ai_ticket_insights` — `_id, group_id, ticket_id, summary, severity_prediction, suggested_fix, classification, created_at, updated_at`
- `ai_group_reports` — `_id, group_id, report_data, generated_at, generated_by`
